终于到了第一个检查点……反正也就是照抄作者的代码，不如学习如何写注释：JsDoc：https://jsdoc.app/index.html

## 目标
* 构建一个接收和投递包裹的邮件发送机器人。
    * 指出从每个地方出发可以到达哪些地方。将字符串表示的连接转化成邻接表。
    * 构建一个表示村庄状态的类，包含当前机器人所在地和留下的包裹，还有一个move方法。
        * 不要为每个概念都编写一个类，只要必要的就可以了。
    * 寻找路线的几种策略：
        - [x] 随机策略
        - [x] 按照固定路线
        - [x] 按顺序递送包裹
        - [x] 每次都去到最近的包裹或者目的地。（贪心算法）
        - [ ] 是不是有更好的路径规划算法？最小生成树？
